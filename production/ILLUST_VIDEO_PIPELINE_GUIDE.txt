 "세계 5개국이 아이에게 AI를 어떻게 가르치나"를, 개념을 은유 일러스트로 그려
 (로컬 AI 이미지 생성) + 한글 타이틀/자막 오버레이 + 내레이션으로 조립한
 2~3분짜리 다큐형 영상. 참고한 스타일 = 한국형 경제/기술 유튜브 explainer
 (100% AI 일러스트 + 장면 내 한글 타이틀 + 하단 자막 + 약한 줌).

■ 2. 전체 파이프라인 (한 번의 스크립트 실행으로 영상 1편)

  [1] 개념 설계         → 섹션 8개(오프닝·중국·미국·영국·싱가포르·한국·공통·결론)
   (사람/Claude가 작성)    각 섹션 = 타이틀 + 서브컷 3개(각각: 영어 이미지 프롬프트 + 한글 자막)
        │
        ▼
  [2] 이미지 생성       → 로컬 ComfyUI(SDXL, RealVisXL_V5.0)로 개념 일러스트 PNG
   (100% 로컬 GPU)         · 1344x768, 텍스트 없이(SD는 한글 못 그림), 장당 약 30초
        │
        ▼
  [3] 한글 오버레이     → Pillow로 "투명 레이어" PNG 생성:
   (로컬)                  액자 비네트 테두리 + 상단 리본 타이틀 배너 + 하단 흰 자막(검은 외곽선)
        │
        ▼
  [4] 모션(켄번즈)      → ffmpeg zoompan으로 "일러스트만" 약하게 줌인(1.0→1.06)
   (로컬)                  그 위에 [3]의 오버레이를 "정적"으로 합성 → 글이 안 잘림
        │
        ▼
  [5] 내레이션          → edge-tts(무료 온라인)로 섹션별 음성 합성 → atempo=1.3(30% 빠르게)
   (온라인 무료)
        │
        ▼
  [6] 조립             → ffmpeg concat: 컷 영상들 이어붙임 + 섹션 음성 이어붙임 + 2초 꼬리
   (로컬)                → 최종 mp4 (1280x720, 30fps, H.264/AAC)


■ 3. 기술 스택 (★로컬 vs 온라인 명확히)
  - 이미지 생성 : ComfyUI 0.22 + RealVisXL_V5.0 (SDXL)  → ★100% 로컬, GPU RTX 3060(12GB), 무료
  - 내레이션    : edge-tts (Microsoft Neural, ko-KR-SunHiNeural) → 온라인 API(무료, 인터넷 필요)
  - 자막/타이틀 : Python Pillow (한글 폰트 Malgun Gothic Bold) → 로컬
  - 영상 조립   : ffmpeg 8.1.1 (zoompan, overlay, concat, atempo) → 로컬
  - 조종(오케스트레이션) : Python 스크립트 1개 → 위 전 과정을 순서대로 자동 실행

  ※ 즉, "돈 드는 유료 AI 이미지 서비스" 없이, 로컬 그래픽카드로 이미지를 뽑는다.
    TTS만 무료 온라인. 나머지 전부 로컬.


■ 4. 단계별 상세 파라미터 (재현용)

 [이미지 프롬프트 구조] (영어로 SDXL에 투입)
   장면묘사 + 고정 스타일토큰
   고정 스타일 = "editorial storybook illustration, warm muted color palette,
     thick clean black outlines, semi-realistic cartoon style, flat shading,
     soft warm lighting, detailed cozy background, hand-drawn look, wide cinematic composition"
   네거티브   = "photo, photorealistic, 3d, text, letters, korean text, watermark, logo,
     deformed, extra fingers, blurry, creepy, surveillance, headband, weapon"
   ※ 스타일토큰을 모든 컷에 동일 적용 → 일관된 톤. 컷마다 seed 고정 → 재현 가능.

 [SDXL 설정] RealVisXL_V5.0_fp16 / 1344x768 / steps 30 / cfg 6.5 /
             sampler dpmpp_2m / scheduler karras / denoise 1.0

 [켄번즈] zoompan z: 1.0 → 1.06 (약하게), 프레임 중앙 기준. ★핵심: 텍스트는 줌에
         포함하지 않고 줌 "위"에 정적 합성 → 확대해도 글자 안 잘림.

 [자막/타이틀] 폭 초과 시 폰트 자동 축소(안전 여백). 하단 자막=흰 글자+검은 8방향 외곽선.
              타이틀=크림색 리본 배너(양끝 접힘), 섹션 내 유지·자막만 컷마다 교체.

 [속도] 내레이션 mp3에 ffmpeg atempo=1.3 (피치 유지한 채 30% 빠르게).
        영상 컷 길이는 "배속 처리된 섹션 오디오 길이 ÷ 컷 수"로 정확히 싱크.


■ 5. 핵심 노하우 (이게 품질의 비결)
 (1) SD는 한글을 못 그린다 → 일러스트는 "텍스트 없이" 생성하고, 한글은 Pillow로 따로
     얹는다. (참고 영상도 같은 구조로 추정) — 우리가 이미 Pillow에 강해서 유리.
 (2) 줌하면 가장자리 글이 잘린다 → 일러스트만 줌, 배너/자막은 줌 위 "정적 오버레이".
 (3) 오디오-영상 싱크 → 섹션별로 음성을 따로 합성해 길이를 알고, 그 길이에 컷을 맞춤
     (음성과 영상을 각각 같은 섹션 순서로 이어붙이면 100% 싱크).
 (4) 톤 일관성 → 스타일 토큰 고정 + seed 고정. 캐릭터 완전 동일화는 아직 미해결(별도 과제).
 (5) 정직성 규칙 → 특정국 비하/감시 프레임 금지(중국 뇌파밴드 등 배제), 수치는 조작 금지.

■ 7. ★자동화 수준 (질문에 대한 정직한 답)
  - "한 번의 스크립트 실행 → 완성 영상"까지는 자동이다.
    (이미지 24장 로컬 생성 → 오버레이 → 켄번즈 → 내레이션 배속 → 조립, 사람 손 없이 진행)
  - 단, "무엇을 그릴지(장면 프롬프트)와 무슨 자막을 넣을지"는 지금은 사람/Claude가
    미리 써 넣는다. 즉 "대본+프롬프트가 준비되면" 그 다음은 완전 자동.
  - 아직 자동 아닌 것: 주제만 던지면 대본·프롬프트·자막까지 스스로 짜는 단계(→ 다음 목표).
  → 요약: "렌더링은 100% 로컬 자동, 기획(대본/프롬프트)은 아직 사람".
     기획 자동화 한 겹만 얹으면 "주제 입력 → 영상 출력"까지 무인화 가능.


■ 8. 확장/개선 로드맵
  - 캐릭터 일관성: IP-Adapter / LoRA로 같은 아이·부모 반복 등장
  - BGM/효과음: CC0 무료 음원으로 긴장감·컷 강조
  - 컷 더 촘촘히: 문장 단위(섹션당 4~5컷)
  - 기획 자동화: 주제 → (LLM이) 대본+장면 프롬프트+자막 생성 → 파이프라인 투입 → 무인 양산
